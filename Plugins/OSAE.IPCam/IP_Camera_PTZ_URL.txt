IP PTZ Camera Database 02/11/2014
Complied by: Kirk Herron
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The following database is a compilation of PTZ IP Camera URL's.

Included with each Brand/Model is:

	Authentication: This is usually needed to authenticate the user accessing the camera.
                                         If this field is blank, try the StreamURLs to see if you get an authentication error.
                                         If you do. try to locate another model of the same brand and see if the autentication described there works with your model.
		         This field may show more than one option. You may need to try them all to find the correct one.

       MJPEG StreamURL : This is the URL used to stream or retrieve a MJPEG image from the camera. This can easily be tested by 
                                         entering this URL in to IE. You should see an image or the video feed from your camera. This list may show more
                                         than one URL. Try them all to find the one that works with your model.

           JPEG StreamURL: This is the URL used to stream or retrieve a JPEG image from the camera. This is ussually used fro Snapshots.
                                         This can easily be tested by entering this URL in to IE. You should see a snapshot image from your camera. This list may
                                         show more than one URL. Try them all to find the one that works with your model.

                CommandURL: This is the URL used to send PTZ or other commands to the camera.
		         
	     Commands: The standard commands available through the use of the CommandURL.
		         Commands with N/A will not function with the selected Brand/Model.

     Extended Commands: Additional Functions or Commands that can be sent to the camera.
		        These may describe the use of a different CommandURL and Params. 
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
NOTES:

As with all IP cameras, you must first include the IP address and the Port number. The following database does NOT describe these as it should 
always to be included in the URL. The proper syntax is as follows: http://ipaddress:port/ (http://192.168.1.101:80/).
This should be included at the beginning of any URL used to communicate with your device.

Fieldnames are surrounded by square brackets [ ]. These are to be replaced with the actual information required. A good example of this is in the
Authentication field. Where you see [USERNAME] and [PASSWORD], you need to enter the actual Username and Password. For example: if the
Authentication shows:  user=[USERNAME]&pwd=[PASSWORD], and your uersname is Admin and your password is password, 
you will need to enter:  user=admin&pwd=password. DO NOT leave the square brackets!
Use any other Fields the same way, as they will be represented with square brackets also.

As with all URL's, you must enter a question mark (?) after the command, then separate all the additional parameters with the AND symbol (&).
For example: /video.cgi?resolution=VGA&user=admin&pwd=password

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++
As an example we will use the first model located below:
MJPEG StreamURL = [IPADDRESS]:[PORT]/videostream.cgi?user=[USERNAME]&pwd=[PASSWORD]
                                    (http://192.168.1.101:80/videostream.cgi?user=admin&pwd=password)

JPEG StreamURL = [IPADDRESS]:[PORT]/snapshot.cgi?next_url=[Filename]&user=[USERNAME]&pwd=[PASSWORD]
		     (http://192.168.1.101:80/snapshot.cgi?next_url=test1.jpg&user=admin&pwd=password)

CommandURL = [IPADDRESS]:[PORT]/decoder_control.cgi?command=25&user=[USERNAME]&pwd=[PASSWORD]
                                     (http://192.168.1.101:80/decoder_control.cgi?command=25&user=admin&pwd=password)

TO USE:
Click Here. Then choose EDIT-->Find.  OR  hit CTRL+F.
Type in your Brand or Model and click Find Next.
Continue to click Find Next or F3 and search again to see if there are more matches.

You may have to try sever entries before you find the one that works for you.
Many cameras use the same firmware, thus, use the same URL's for controls.

This compilation is from many different sources and may contain typo's or incorrect information.
There is no guarentee that the information here will work with your device!
